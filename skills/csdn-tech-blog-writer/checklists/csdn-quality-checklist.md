# CSDN 技术博客质量检查清单

生成博客前，请检查以下内容：

## 一、代码分析

- [ ] 是否读取并分析了用户提供的核心代码？
- [ ] 是否识别了项目技术栈？
- [ ] 是否分析了 Controller、Service、Mapper 等核心层？
- [ ] 是否分析了实体类和数据库字段？
- [ ] 是否分析了配置文件？
- [ ] 是否排除了无关目录和构建产物？

## 二、内容真实性

- [ ] 是否没有编造项目中不存在的功能？
- [ ] 是否没有编造接口路径？
- [ ] 是否没有编造数据库表？
- [ ] 是否没有编造运行截图或运行结果？
- [ ] 如果是推断内容，是否明确说明？

## 三、代码规范

- [ ] 代码块是否标注语言类型？
- [ ] 代码块是否标注文件路径？
- [ ] 核心代码是否添加中文注释？
- [ ] 是否只展示关键代码，而不是无意义地粘贴大段代码？
- [ ] 示例代码是否明确标注为示例？

## 四、CSDN 格式

- [ ] 是否使用 Markdown 格式？
- [ ] 标题层级是否清晰？
- [ ] 是否适当使用表格？
- [ ] 是否适当使用列表？
- [ ] 是否适当使用引用块？
- [ ] 是否可以直接复制到 CSDN 编辑器？

## 五、原创性

- [ ] 是否没有复制其他博客内容？
- [ ] 是否没有洗稿？
- [ ] 是否结合用户自己的代码进行讲解？
- [ ] 是否有自己的分析和总结？

## 六、安全与隐私

- [ ] 是否隐藏数据库密码？
- [ ] 是否隐藏 Token、SecretKey、AccessKey？
- [ ] 是否隐藏真实手机号、邮箱、身份证号？
- [ ] 是否没有泄露服务器 IP、生产环境地址等敏感信息？

## 七、文章质量

- [ ] 前言是否说明文章价值？
- [ ] 技术点讲解是否清楚？
- [ ] 代码和文字是否对应？
- [ ] 是否包含注意事项？
- [ ] 是否包含优化建议？
- [ ] 总结是否有实际信息量？
- [ ] 标签是否与文章内容相关？

## 八、用户风格一致性

当用户提供已有博客或要求保持个人风格时，请检查：

- [ ] 是否阅读了用户已有博客内容？
- [ ] 是否分析了用户标题风格？
- [ ] 是否分析了用户开头习惯？
- [ ] 是否分析了用户章节结构？
- [ ] 是否分析了用户正文表达习惯？
- [ ] 是否分析了用户代码讲解方式？
- [ ] 是否分析了用户结尾风格？
- [ ] 新内容是否延续了用户原有语气？
- [ ] 新内容是否延续了用户原有标题层级？
- [ ] 是否避免重复用户已写过的内容？
- [ ] 是否没有直接复制历史博客中的大段原文？
- [ ] 是否在保持风格的同时保证技术准确性？
- [ ] 如果修改了用户原文，是否没有把文章改得不像用户本人写的？
